## 取得不可能なデータ（API仕様による制限）

| データ項目 | 理由 |
|----------|-----|
| ユーザーが入力したプロンプト内容 | プライバシー保護 |
| Copilotが生成したコード内容 | プライバシー保護 |
| 操作対象のリポジトリ名 | プライバシー保護 |
| 操作対象のファイルパス | プライバシー保護 |
| GitHub.com上のCopilot Chat | Usage Metrics対象外 |
| GitHub Mobile利用状況 | Usage Metrics対象外 |
| Copilot CLI利用状況 | Usage Metrics対象外 |
| Copilot Code Review利用状況 | Usage Metrics対象外 |

## 現在のスクリプトで取得しているデータ

### 取得済み（Sheet 1: メインレポート）

- ユーザー名、参加日、Copilot有効/無効
- シート割り当て、最終利用日時、最終利用エディタ
- シート作成日、シート取消予定日

### 取得済み（Sheet 2: 組織全体の統計）

- 日別のアクティブユーザー数・エンゲージユーザー数
- コード補完（利用者数、提案数、承認数）
- IDE Chat（利用者数、メッセージ数、挿入数）
- 28日間のサマリー統計

### 取得済み（Sheet 3: メトリクス項目説明）

- 各メトリクスの説明
- 重要なポイント
- データ概要

### 未実装（取得可能だが実装していない）

- 言語別・エディタ別・モデル別の詳細内訳
- GitHub.com Chat統計
- Pull Request機能の統計
- Chatコピーイベント数
- チーム別メトリクス

### 実装したが動作しない（API制限）

- ユーザー別のプロンプト数・生成数・受け入れ数
- ユーザー別の追加LoC・アクティブ日数・主要IDE
  → これらは全て0（Usage Metrics APIが404）

## 推奨される今後の拡張

### 優先度：高

1. **言語別・エディタ別の内訳を追加**
   - 既存APIレスポンスに含まれている
   - Sheet 2に追加テーブルとして実装可能

2. **チーム別メトリクスの取得**
   - `/orgs/{org}/team/{team_slug}/copilot/metrics`
   - 部門別の利用状況分析に有用

### 優先度：中

3. **GitHub.com ChatとPR機能の統計**
   - 既存APIレスポンスに含まれている
   - 組織全体の利用傾向把握に有用

### 優先度：低

4. **Usage Metrics API（NDJSON）の調査**
   - Enterprise Cloudへのアップグレード検討時
   - ユーザー別詳細データが必要になった場合
