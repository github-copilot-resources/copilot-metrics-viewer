# Copilot Metrics Viewer

Visualize, analyze, and understand GitHub Copilot adoption across your organization or enterprise with an interactive analytics dashboard.

**Version 3.2.0** · [Source Code](https://github.com/github-copilot-resources/copilot-metrics-viewer) · [Deployment Guide](https://github.com/github-copilot-resources/copilot-metrics-viewer/blob/main/DEPLOYMENT.md) · MIT License

---

## What It Does

Copilot Metrics Viewer is a self-hosted web application that connects to the [GitHub Copilot Usage Metrics API](https://docs.github.com/en/enterprise-cloud@latest/rest/copilot/copilot-usage-metrics) and presents your Copilot data through interactive charts, tables, and summaries. It is a **read-only** dashboard — it never modifies your repositories, code, or settings.

## Key Features

### 📊 Comprehensive Metrics Dashboard
- **Acceptance rate** — ratio of accepted suggestions to total suggestions (inline code completions)
- **Suggestions & lines of code** — total suggested vs. accepted, tracked over time
- **Active users** — daily active Copilot users with trend visualization
- **Language breakdown** — top languages by accepted prompts and acceptance rate

### 💬 Copilot Chat & Agent Activity
- Chat interaction metrics: turns, acceptances, active users
- Agent activity: code edits, completions, and lines of code generated by Copilot agents
- Pull request metrics: PRs created, reviewed, and merged with Copilot assistance

### 👥 Per-User Metrics
- Individual usage breakdowns including code completions, chat, and agent activity
- **Copilot LOC** — total lines of code generated per user with per-feature breakdown
- Summary tiles: total users, active users, average acceptance rate
- Educational info panel explaining metrics and adoption patterns

### 🏢 Teams Comparison
- Compare Copilot adoption across teams within your organization
- Identify high-performing teams and share best practices
- Team metrics derived automatically from per-user data and GitHub team membership

### 📈 Historical Mode (Optional)
- Retain metrics beyond the 28-day API rolling window using PostgreSQL
- Per-user time-series history with trend charts
- Full historical team analytics
- Automated daily sync service

### 🤖 AI-Powered Chat (Optional)
- Ask natural language questions about your Copilot metrics
- Powered by [GitHub Models API](https://docs.github.com/en/github-models) (gpt-4o)
- Analyzes only the metrics data already visible on your dashboard
- Can be enabled/disabled via configuration

### 🗓️ Flexible Date Range
- Filter metrics for custom date ranges (up to 100 days)
- Option to exclude weekends and holidays

### 📤 Data Export
- CSV export (summary and full detail)
- Clipboard copy for quick sharing
- Data quality validation

## Required Permissions

### GitHub App Permissions

When installed as a GitHub App, the following **organization permissions** are required:

| Permission | Access | Purpose |
|---|---|---|
| **Members** | Read-only | Resolve team membership for team-scoped metrics |
| **Copilot Metrics** | Read-only | Fetch Copilot usage metrics data |
| **Copilot Seat Management** | Read-only | Fetch seat assignment and activity data |

### Personal Access Token (Alternative)

If using a fine-grained Personal Access Token instead of a GitHub App, the token needs the same organization permissions listed above.

> **Important:** The token or GitHub App must have access to the target organization. Enterprise-scoped metrics require a token with enterprise access.

---

## Supported Scopes

| Scope | URL Pattern | Description |
|---|---|---|
| Organization | `/orgs/{org-name}` | Metrics for a single GitHub organization |
| Enterprise | `/enterprises/{enterprise-name}` | Metrics across an enterprise account |
| Team (Org) | `/orgs/{org-name}/teams/{team-name}` | Team-level metrics within an organization |
| Team (Enterprise) | `/enterprises/{enterprise-name}/teams/{team-name}` | Team-level metrics within an enterprise |

## Deployment

Copilot Metrics Viewer is **self-hosted** — you deploy it on your own infrastructure. No data leaves your environment except API calls to GitHub.

### Quick Start Options

1. **☁️ One-Click Azure Deployment** — Azure Container App + PostgreSQL (~$15/month)
2. **🐳 Docker / Docker Compose** — Run anywhere with a single container
3. **☸️ Kubernetes** — Helm-compatible manifests included
4. **💻 Local Development** — `npm install && npm run dev`

See [DEPLOYMENT.md](https://github.com/github-copilot-resources/copilot-metrics-viewer/blob/main/DEPLOYMENT.md) for detailed setup instructions.

### Authentication Options

| Method | Description | Enterprise Support |
|---|---|---|
| **GitHub App OAuth** (Recommended) | Users authenticate with their GitHub credentials; only users with appropriate org permissions can view metrics | Organization only |
| **Personal Access Token** | Server-side token, no per-user authentication | Organization & Enterprise |
| **GitHub OAuth App** | For enterprise-level authentication | Organization & Enterprise |

## Data & Privacy

- **Read-only**: The application only reads metrics from GitHub APIs — it never writes or modifies anything
- **Self-hosted**: All data stays within your infrastructure
- **No telemetry**: The application does not collect usage analytics or phone home
- **No third-party data sharing**: Data is only sent to GitHub APIs (and optionally GitHub Models for AI chat)
- **Session encryption**: User sessions are encrypted with a configurable secret
- **Open source**: Full source code available for security audit under MIT license

See [TRANSPARENCY_DISCLOSURES.md](https://github.com/github-copilot-resources/copilot-metrics-viewer/blob/main/TRANSPARENCY_DISCLOSURES.md) for detailed security and compliance information.
